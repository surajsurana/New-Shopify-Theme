## 1. Audit Summary (Phase 1)

**Current live page (`karishmaashita.com/pages/about-us`), verified directly and cross-checked against two independent third-party sources (see Section 5):**

The page contains real, substantive, non-generic brand copy — this is not a blank or default Shopify page. It has: a founder bio ("K&A is a Mumbai-based luxury fashion label specialising in bespoke bridal couture and premium ready-to-wear"), founder names (Karishma Surana and Ashita Surana, "best friends turned co-founders"), a philosophy statement ("every bride deserves a garment made with intention"), named craft techniques (zardozi, threadwork, sequin embellishments), and service facts (Mumbai flagship store, virtual appointments, complimentary worldwide shipping, DHL Express).

**But it is presented with zero art direction.** Based on the page's structure — an announcement-bar strip, then a bio paragraph, a bulleted "Key Offerings" list, a "Design Philosophy" paragraph, and a "Quick Facts" block — this reads as a flat, SEO-style spec sheet, not an editorial brand story. There is no hero image, no founders imagery, no craft photography, no pacing, no emotional arc. It is the textual equivalent of a product spec table applied to the two people who built the entire house.

**Benchmark comparison:** Sabyasachi's "The House" page, Gaurav Gupta's "About," Dior's "La Maison," and Chanel's "Gabrielle Chanel" pages all do the same thing structurally: they open with a single cinematic image, introduce the founder(s) as a headline-worthy name rather than a bio-paragraph fact, let the philosophy breathe as an isolated quote rather than a bullet point, and close by inviting the visitor into the atelier. K&A's real, true content already contains everything needed to do this — it has simply never been given the register the homepage already established. The gap is not missing facts; it is missing craft.

**Emotional audit of current live page:** Indifference. A visitor learns facts but feels nothing — no sense of the two women behind the house, no sense of the hands that make the clothes, no invitation to imagine themselves in the atelier. This is the single largest gap this redesign closes.

**Emotional audit of redesigned page:** Trust, moving toward Romance. The page now opens on the house's own name as a headline, narrates the founding as an intimate, specific story rather than a bio-paragraph, gives the craft physical specificity (zardozi, boning, sequins named and shown, not just claimed), and closes on the same appointment invitation used sitewide — so trust converts directly into the next real action (booking).

## 2. Redesign Concept (Phase 2)

**Emotional arc:** Cinematic hero, the house's own name treated as headline (silence, one image, one line) → an intimate origin story told as narrative, not bio-copy → a single isolated vision statement, given room to breathe → a dark, dramatic craft interlude that makes the handwork visually specific rather than merely claimed → a quiet reinforcement of the trust facts already established sitewide (Mumbai atelier, made-to-order, worldwide delivery, concierge) → a quiet, text-only note that the house's footprint keeps growing beyond Mumbai → the same closing appointment invitation that already lives on the homepage, because About's job is to build trust and then hand off to the exact same conversion moment, not invent a new one.

**Structural decision — reuse over invention:** Five of this page's seven content sections are direct reuses of already-approved homepage components (Hero, Manifesto → Vision, Craft Strip, Appointment CTA, Footer, Nav). Two things are genuinely new from the original v1.0 approval: the Founders/Origin split panel (an adapted, simplified variant of the homepage's Collection Story component) and the Craft & Process dark diptych (a new arrangement of an existing hover-caption pattern borrowed from the homepage Lookbook). This is a deliberate choice, not a shortcut — a page whose entire purpose is to prove "this is one coherent, considered house" should visually *feel* like the same house at every scroll, not introduce a parallel design language. Reusing the exact same component vocabulary is itself part of the luxury argument. **Added in v1.1:** a seventh section, Stockist Footprint (new, text-only, dark ground — see Section 3), built once Suraj confirmed the stockist list should be shown; it follows the same reuse discipline, borrowing the Craft & Process header pattern and the Manifesto's hairline-rule instinct rather than inventing new visual language for a single new section.

**Why no literal "team" or "founder portrait" section:** No real photograph of Karishma or Ashita exists in the assets gathered. Rather than fabricate a "meet the founders" moment with stock/model photography captioned with their real names (which would be a trust-destroying move the moment anyone who has met them in person or seen a real photo notices the mismatch), the founders' story is told entirely through copy, paired with editorial imagery used honestly — as mood and craft, never as documentary evidence of two specific individuals. See Section 5 for exactly what real photography is needed to complete this properly.

**Buy path stays reachable**, per standing principle: hero carries both "Explore the Collections" (commerce) and "Book a Styling Appointment" (concierge) with equal visual weight, and the page closes on the identical dual-CTA appointment block used on the homepage. About Us is not a dead-end brand essay — it is a trust-building bridge that hands off to the same two real actions available everywhere else on the site.

**Pacing follows the established light/dark rhythm** (per homepage-build-spec.md Section 9, Note 12): Hero (dark) → Founders (light) → Vision (light) → Craft & Process (dark) → Atelier & Trust Strip (dark) → Stockist Footprint (dark) → Closing CTA (dark image) → Footer (dark). Four consecutive dark sections at the close mirrors the homepage's own precedent even more closely (which closes with six consecutive dark sections) — the new Stockist Footprint section was placed here, rather than broken out as a light section, specifically to continue this already-validated cadence rather than interrupt it.

## 3. Section-by-Section Breakdown

### Section 1 — Navigation
Identical global nav/mobile-nav component used on the homepage. No changes. The mobile nav drawer's "About" link now points to `/pages/about-us` and carries an `.is-current` state (new, minor CSS addition — colour matches rose-gold hover state, see Section 4).

### Section 2 — Hero
**Purpose:** Full-viewport cinematic open. The house's own name is the headline — not a slogan, not a claim, the name itself given the same visual authority the homepage gives "Bridal / Couture."
**Structure:** Full-bleed image (`about-hero.jpg`), gradient overlay (slightly heavier than homepage's hero overlay — `0.30/0.08/0.68` vs `0.22/0.05/0.62` — because this hero is shorter at `92svh` rather than full `100svh` and needs marginally more contrast to keep white text legible against a lighter part of the frame), eyebrow ("The House, Mumbai — Est. 2016" — **updated in v1.1**; the founding year is now confirmed by Suraj and appended to the existing eyebrow phrase using the same em-dash-plus-"Est." construction already proven live on the homepage hero, "Mumbai — Est. in Craft," rather than inventing a new phrasing pattern), H1 "Karishma / & Ashita" (same large-italic + `<strong>` sub-line split used on the homepage's "Bridal / Couture," reapplied here to name the founders directly rather than describe the product), subtitle stating the real, verified philosophy in first-person house voice, two CTAs, scroll indicator.
**Height decision:** `92svh` rather than the homepage's full `100svh` — this hero is intentionally slightly shorter so the Founders section below feels like an immediate continuation of the same thought, not a fully separate "next chapter." A minor, deliberate pacing choice.
**Motion:** Identical fadeUp/fadeIn staggered sequence to homepage hero. No parallax on this page (see Section 4).
**Image used:** `about-hero.jpg` — three models in an editorial group portrait, gold and sage lehengas. Used as an opening mood statement (the world of K&A) rather than a literal illustration of "two founders."

### Section 3 — Founders / Origin
**Purpose:** Tell the real founding story as intimate narrative, not bio-copy — now paired with a real photograph of the two people telling it.
**Structure:** Adapted from the homepage's Collection Story component — same `grid-template-columns: 1fr 1fr`, same `--c-paper-deep` text panel, same typographic scale (`--t-display` italic H2, `--t-body-l` body). The homepage version's secondary two-image strip beneath the text is deliberately omitted here — only one founders photograph is being used, and manufacturing a second image purely to match a different section's grid would dilute rather than add.
**Caption added in v1.2.** The image now carries a discreet name caption (`.founders__image-caption` — bottom-left, over a soft bottom-up gradient, `.founders__image-overlay`) reading "Karishma & Ashita / Founders, K&A", styled in the same small-italic-display-plus-uppercase-sub pattern already used for `.craft-story__caption-name/__sub`. This is a deliberate reversal of v1.0/v1.1's "no caption" rule — that rule existed specifically because the photo was symbolic model photography; it does not apply to a real, identifiable photo of the two founders themselves, and captioning it now is the more honest choice, not a stylistic embellishment.
**Copy:** Original, house-voice narrative built from three verified facts (see Section 5, "Verified Sources"): the founders' names and "best friends turned co-founders" relationship, the "made with intention" philosophy line, and the Mumbai-atelier/made-to-order operating model. Deliberately echoes but does not repeat verbatim the homepage manifesto's "every lehenga, every corset, every draped silhouette" phrase, so the two pages feel like one continuous voice without being redundant if a visitor reads both in one session. Copy itself is unchanged in v1.2 — only the image and caption changed.
**Image used (v1.2):** `about-founders-real.jpg` — Karishma and Ashita together, professionally photographed in a soft-lit garden setting, both in hand-embellished K&A lehengas (deep red-and-gold and chartreuse-and-gold), warm direct expressions, uncluttered bokeh background (garden wall, fence, foliage). Selected over the other two real candidates supplied (see Section 5) for being the only one that reaches genuine editorial register — the other two are candid event snapshots (one with both founders in sunglasses, obscuring their expressions; one with a busier, crowded event background) that read as personal photography rather than a considered "meet the founders" moment. A "Dream Diaries" third-party photographer watermark sat across the top-left ~190px of the original frame; cropped out entirely (crop box: full width, top 190px removed, final image 1066×1410px) rather than blurred or stamped over, so no trace of a third party's mark remains in the published asset. `object-position` on this image is set to `center 14%` (vs. the previous `22%`) to keep both faces — which sit slightly higher in this frame than in the old placeholder — properly centred once the container crops for `cover`.

### Section 4 — Vision
**Purpose:** A single isolated brand-philosophy statement, given room to breathe, mirroring the homepage Manifesto's function exactly (same component, reused verbatim).
**Structure:** Centred quote, hairline rules before/after, attribution line. Identical CSS class (`.manifesto`) to the homepage — literally the same reusable component, new copy only.
**Copy:** Original house-voice copy inspired by the verified vision statement found via K&A's own retail listing on Pernia's Pop-Up Shop (see Section 5) — not lifted verbatim, since that text is a retailer's third-party marketing description of K&A, not K&A's own first-person brand voice. Rewritten here as an original sentence carrying the same three ideas (opulence, grace, timelessness) in K&A's own register.

### Section 5 — Craft & Process
**Purpose:** Make the handwork specific and visible, not just claimed. This is the section doing the most net-new emotional work on the page — it is where "handcrafted" stops being a marketing word and becomes a boned seam, a placed sequin, a named technique.
**Structure — new:** Dark charcoal ground (`.craft-story`). Centred header block (eyebrow, italic H2 reusing the exact type scale of the homepage's Collections Directory on-dark H2, body paragraph) followed by a full-width two-image diptych at `min-height: 560px` per image. Each image carries the exact hover-reveal pattern from the homepage Lookbook (dark gradient overlay fades in, caption slides up 8px→0) — renamed to this section's own scoped class names (`.craft-story__item/__overlay/__caption*`) per the project's section-scoped CSS convention, but functionally and visually identical.
**Copy:** Names three real, verified techniques (zardozi, threadwork, sequin embellishment, hand-boned corsetry) without inventing false-precision claims (no fabricated hour-counts or years-of-experience numbers — see Section 5 for why).
**Images used:** `about-craft-detail-1.jpg` (structured corsetry, cobalt silk, dramatic single-light studio setup) captioned "Structured Corsetry / Boned and fitted, entirely by hand"; `about-craft-detail-2.jpg` (hand-placed pearl and sequin embellishment on ivory bodice, soft curtained backdrop) captioned "Hand-Placed Embellishment / Zardozi, threadwork, sequins."
**Added in v1.1 — celebrity credit line:** a new short paragraph (`.craft-story__credit`), sitting directly beneath the existing, untouched `.craft-story__body` copy, reads: *"The same hands have, on occasion, dressed private clients across film and fashion — among them actress Palak Tiwari, styled in one of our hand-cut corset drape sarees."* Styled smaller and more muted than the body copy (0.78rem, italic, 42% white) so it reads as a quiet citation rather than a competing claim. See Section 5 for exactly what was verified and why this specific name and phrasing were used.
**Filter note:** These two images use a lighter sepia value (`0.05`/`0.10`) than most homepage imagery (typically `0.10`–`0.12`) specifically to preserve the saturated cobalt blue in `about-craft-detail-1.jpg`, which is a deliberate, striking colour moment in the original photography. See Section 4 (tokens) for the exact values and rationale — this is a documented, deliberate per-image exception, not scope creep on the "the filter is non-negotiable" rule from homepage-build-spec.md Section 9 Note 1 (the filter is still applied everywhere; only its exact strength varies, exactly as that same note already permits: *"The exact values vary slightly by section and are documented above — use the correct value for each context"*).

### Section 6 — Atelier & Trust Strip
**Purpose:** Reinforce the same operational trust facts already established sitewide (Mumbai atelier, made-to-order, worldwide delivery, concierge), in the exact visual language the homepage already uses for this — a signal of consistency, not a new claim.
**Structure:** Verbatim reuse of the homepage's Craft Strip component (`.craft-strip`) — same class, same dark ground, same four-item layout with rose-gold dot dividers. Only the copy differs, rephrased (not duplicated verbatim) for this page's context: "Mumbai Atelier / Private consultations, always by appointment," "Made to Order / Every piece begins with you," "Worldwide Delivery / Complimentary, wherever you are," "Concierge / A conversation, not a call centre."

### Section 7 — Stockist Footprint (NEW in v1.1)
**Purpose:** Show the real, growing retail footprint beyond K&A's own Mumbai atelier and direct WhatsApp-concierge channel — per Suraj's explicit decision to display this (reversing v1.0's "flagged as a decision, not added unilaterally" holding position). Functions as a third-party credibility marker sitting between the sitewide operational trust facts (Atelier & Trust Strip, immediately above) and the closing appointment invitation.
**Structure — new:** Dark charcoal ground (`.stockists`), matching Craft & Process and the Atelier & Trust Strip either side of it — this keeps the page's closing run at four consecutive dark sections into the footer, echoing the homepage's own six-consecutive-dark precedent even more closely than v1.0 did (see Section 2). Centred header block (eyebrow "Beyond the Atelier," italic H2 "A House That Travels.", body paragraph) above a two-column list (`.stockists__grid`) — Online / In Store — separated by a hairline top border, stacking to one column on mobile. Deliberately text-only: no photography exists of these stockists' own retail spaces, and manufacturing imagery here would overstate a footprint that is real but secondary to the made-to-order/concierge model that remains this house's primary story. No third-party logos or borrowed retailer branding — store names are set in the house's own italic Cormorant Garamond, keeping the visual authorship K&A's own even while naming other retailers.
**"Always expanding" framing:** Suraj was explicit that this list keeps growing and should never read as finished. This is carried in three places: the body copy ("This footprint is never finished; we're always in conversation with new cities, new stores, new borders."), and a small "And more, always expanding" line closing out each of the two columns — so the ongoing/expanding nature isn't a single buried sentence but a framing that repeats at both the section and column level.
**Copy — house-voice, not pasted verbatim:** Suraj's supplied information ("we are not only present in Online in Pernia, Aza, Nikaza, Agashe and more. Also we are present in Offline in Anantam - Delhi, Almari - Hyderabad...") was rewritten into the page's existing register rather than quoted directly — matching the same standard already applied to every other section on this page (see Section 5, "Verified real content used in this build").
**Content used:**
- *Online:* Pernia's Pop-Up Shop, Aza, Nikaza, Agashe.
- *In Store:* Anantam (Delhi), Almari (Hyderabad), Silk & Threads (Vancouver, Canada), Nezo (Bandra, Mumbai).
- Both lists sourced verbatim (names/cities only, not phrasing) from Suraj directly on 11 July 2026 — not independently re-verified against each retailer's own site, since this is first-party information from the brand's own co-founder/operator, not a claim requiring third-party corroboration the way the celebrity credit did.
**Images used:** None — deliberately (see Structure above).

### Section 8 — Closing CTA (Appointment)
**Purpose:** The page's single macro conversion moment. Reuses the homepage's exact Appointment CTA component (`.appointment`) — same class, same dual `.btn-outline-white` CTA pattern, same "or" divider — because About's job is to build trust and then route directly into the same booking flow used everywhere else on the site, not invent a separate one.
**Copy:** New heading ("Every K&A Story / Starts the Same Way.") and new body copy, close in register to the homepage's "Begin with a Conversation" but not identical, so a visitor who has seen both pages in one session doesn't experience literal repetition.
**Image used:** `about-closing-cta.jpg` — a corset lacing detail, warm-lit, intimate. Chosen deliberately over a generic "atelier interior" or "storefront" shot: it is the single most literal visual expression of "made for you" available among the five sourced images — the fitting itself, not a room.

### Footer
Shared global footer — full specification lives in `Docs/homepage-build-spec.md` Section 13 and is not re-litigated here. The "About Karishma & Ashita" footer link now carries an `.is-current` state on this page (new, minor CSS — see Section 4).

Note that `Docs/homepage-build-spec.md` and `Prototypes/homepage-v2.html` still reference Bodoni Moda themselves — that is stale documentation from an earlier iteration (theme.liquid's own code comment cites "spec v2.0" as the point the font changed to Cormorant Garamond) and should not be treated as the standard; the live build is the source of truth.

Two type-scale values were also tuned after the font swap, since Cormorant Garamond's more open italic letterforms sit slightly differently than Bodoni Moda's at very large display sizes — not new tokens, just bringing this page's two large italic headlines in line with line-height values already proven safe in production for the identical font/weight/style/size elsewhere on the site:
- `.hero__title` line-height: `0.94` → `0.96`, matching Client Diaries' `.diary-hero__title` (same `var(--t-hero)` size, same italic Cormorant Garamond) exactly.
- `.manifesto__line` line-height: `1.35` → `1.4`, matching Client Diaries' `.diary-intro__line` (the same pull-quote component pattern, also italic Cormorant Garamond) — this page's version runs to a slightly larger max size (`3rem` vs `2.6rem`), so if anything it needed the same or more room, not less.

All other italic display headlines on this page (`.founders__title` 1.05, `.craft-story__title` 1.1, `.appointment__title` 1.05) were checked against equivalent live Cormorant Garamond headlines elsewhere (`.section-header__title` 1.05, `.diary-cta__title` 1.08) and already sit within the same safe range — no changes needed there.

## 6. Shopify Platform Feasibility Flags

### H1 — Page as a Shopify `page.about-us` template vs. a custom section-based page
**Issue:** `/pages/about-us` already exists as a live Shopify Page today (confirmed — it has real content, it is not a 404 or blank page). The redesign replaces flat page-body text with a fully section-based, editorial layout.
**Recommendation:** Build this as a dedicated `templates/page.about-us.json` referencing new sections (`ka-about-hero`, `ka-about-founders`, `ka-about-vision`, `ka-about-craft`, `ka-about-stockists` (new in v1.1), plus reuse of the existing `ka-craft-strip`/equivalent and `ka-appointment-cta`/equivalent sections if those already exist from the homepage build, or new ones scoped identically). This is the standard Shopify pattern for giving one specific Page a fully custom, non-default layout without affecting the generic `page.json` template other pages use. `ka-about-stockists` should expose its two column lists as repeatable blocks (name + city as separate settings) in the Shopify schema, not hardcoded text, so Suraj can add new stockists himself going forward without a code change — directly serving the "always ongoing/expanding" nature he specified.
**Risk:** Low. Same pattern already used successfully for Client Diaries (`templates/page.client-diaries...json` per that page's build).

### H2 — Reused sections (Craft Strip, Appointment CTA) should be the SAME section, not copies
**Issue:** The Atelier & Trust Strip and Closing CTA sections in this page are visually and structurally identical to homepage sections already built (per Section 5 of this document — reuse, not reinvention).
**Recommendation:** If the homepage's Craft Strip and Appointment CTA are already built as standalone, reusable Shopify sections (not homepage-only inline markup), the Technical Director should add them to this page's template with new setting values (different copy, different image) rather than duplicating the section file. If they were built homepage-only, this is the moment to generalize them into reusable sections — worth doing once now rather than duplicating section code across every future page that needs an appointment CTA (Contact, Appointment page itself, etc. are all still upcoming in the build order).
**Risk:** Low technically; moderate in importance — getting this reuse right now saves real duplicated-code risk across the three still-upcoming pages (Appointment, Contact, Footer-adjacent work) that will likely want this exact same CTA block.

### H3 — `.is-current` nav/footer state
**Issue:** New minor state class needs a Liquid condition (`{% if request.page_type == 'page' and page.handle == 'about-us' %}` or equivalent `request.path` comparison) to apply correctly, and ideally should extend to Collection/Product/Cart pages' own nav links too, not just this page.
**Recommendation:** Implement as a small shared Liquid snippet/pattern usable by any page template, not hardcoded per-page. Low complexity, but worth doing once, correctly, rather than as a one-off hack on this page.
**Risk:** Low.

### H4 — Image filter variance per photograph (F11 precedent from homepage spec)
**Issue:** Same platform note as homepage-build-spec.md F11 — inline `style="filter:..."` attributes need to become CSS classes (not inline styles) when images are rendered via Liquid `image_picker` settings, since merchant-uploaded images via the theme editor don't carry arbitrary inline styles automatically.
**Recommendation:** Extend the existing `.img-editorial` class pattern (or equivalent) established for the homepage build with the two new per-image variants introduced here (the lighter-sepia craft-detail treatment). Same solved problem, same solution, two new class variants.
**Risk:** Low.
